## 4. Quyết định thiết kế đáng nhớ

1. **Phân nhánh theo 2 cờ trong localStorage** — `onboardedAt` (đã điền info) và `recommendations.length` (đã có lộ trình). Đơn giản, backend sau này thay bằng session thật mà không đổi cấu trúc UI.
2. **Lộ trình để trống sau onboarding** (quyết định của user) → cần đường demo cho trạng thái "đầy": thêm nút **"Dùng tài khoản demo có sẵn lộ trình"** ngay màn onboarding — một click ra đúng screenshot Figma.
3. **Giữ `lib/interview.ts` (STEPS + `recommend()`)** dù xoá ChatPanel — vẫn được ProfileModal/demoProfile dùng, và là interface cho backend "bài test" thật.
4. **Port wireframe sang theme app** thay vì bê nguyên màu xanh dương của file HTML — giữ nhất quán brand teal `#007C76`.
5. **Chat stub có cấu trúc `reply(text, profile) → {text, sources, quicks}`** — backend LLM + SSE chỉ cần thay một hàm.

## 7. Ghi nhận về cách phối hợp người–AI (dùng được cho AI Log)

- **Pattern tốt:** user chặn AI làm sớm ("đừng làm gì tới khi tôi nói") → AI đọc docs trước; khi yêu cầu mơ hồ ở điểm rẽ nhánh sản phẩm, AI hỏi 3 câu chốt rồi mới code — tránh rework giữa hackathon.
- **AI tự quyết phần còn lại** (theme, cấu trúc component, demo seed) và nêu rõ các quyết định + điểm hở sau khi xong — user chỉ cần phản hồi những gì lệch ý.
- **Mức can thiệp:** AI viết ~100% code mới trong phiên, dựa trên component có sẵn của team; user quyết 3 quyết định sản phẩm + ra lệnh dọn dẹp. Toàn bộ code đã được verify chạy thật trước khi báo xong.
